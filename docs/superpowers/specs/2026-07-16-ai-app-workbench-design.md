# AI 应用工作台设计说明

## 目标

在现有 Vue 3 前端中补齐网站应用的创建、生成、预览、部署、个人管理和管理员管理流程。界面采用“混合式产品工作台”：主页和生成页使用定制化产品界面，管理表格与编辑表单沿用 Ant Design Vue 和现有用户管理页的交互模式。

## 范围

本次前端交付包含：

- 主页提示词创建区、我的应用分页列表和精选应用分页列表。
- 应用生成对话页，包括 SSE 流式回复、生成结果预览和部署。
- 应用信息修改页，包括普通用户与管理员的不同字段权限。
- 管理员应用管理页，包括查询、详情、编辑、删除和精选。
- 应用领域类型、请求服务、SSE 客户端、权限路由和关键逻辑测试。

本次不修改后端实现，不实现文件上传和提示词优化，不复制参考图中的品牌、文案或编辑器功能。参考图只用于布局、信息层级与交互密度参考。

## 设计方向

### 视觉语言

- 主体采用白色和冷灰色，使用清晰的深色文字与青蓝色强调色。
- 主页以提示词输入为首要视觉信号，不使用营销式大卡片或冗长功能介绍。
- 应用列表使用稳定尺寸的封面网格；真实封面优先，无封面时展示与应用名称相关的低噪声占位背景。
- 对话页是沉浸式工作台，移除普通页头和页脚，保持左侧对话、右侧预览的高信息密度结构。
- 管理页继续使用现有用户管理页的筛选栏、表格、分页、抽屉和反馈样式。
- 动效仅用于页面进入、流式生成状态和按钮反馈，并支持 `prefers-reduced-motion`。

### 响应式行为

- 桌面主页应用卡片按可用宽度排列，移动端退化为单列。
- 对话页在宽屏采用左右分栏；窄屏使用“对话 / 预览”页签切换，避免两个面板同时挤压。
- 管理表格允许横向滚动，筛选项在移动端纵向排列。
- 所有输入、按钮、卡片标题和分页控件在 320、375、414 和 768 像素宽度下不得溢出或遮挡。

## 页面与路由

| 路由 | 页面 | 权限 | 布局 |
| --- | --- | --- | --- |
| `/` | 主页 | 公开；个人列表与创建功能需要登录 | 标准布局 |
| `/app/:id/chat` | 应用生成对话页 | 登录用户；生成和部署由后端校验所有权 | 沉浸式布局 |
| `/app/:id/edit` | 应用信息修改页 | 登录用户；管理员可编辑任意应用 | 标准布局 |
| `/app/manage` | 应用管理页 | 仅管理员 | 标准布局 |

路由元信息新增：

- `requiresAuth`：未登录时跳转登录页，并携带原目标地址。
- `requiresAdmin`：沿用现有管理员判断。
- `immersive`：由布局组件隐藏全局页头、页脚和标准内容宽度约束。

## 组件边界

### 应用领域层

- `types/app.ts`：维护 `AppVO`、分页对象和所有应用请求类型。
- `services/app.ts`：封装普通 JSON 接口。
- `services/appSse.ts`：只负责创建、关闭和解析 SSE 连接，将数据片段、完成与错误事件暴露给页面。
- `utils/app.ts`：分页参数规范化、预览 URL 构建、日期与占位信息等纯函数。

### 共享界面组件

- `AppCard.vue`：应用封面、名称、作者、创建时间和应用操作。
- `AppListSection.vue`：标题、名称查询、加载、空状态、网格和分页。
- `PromptComposer.vue`：主页及对话页可复用的多行提示词输入和提交状态。
- `AppDetailDrawer.vue`：管理页查看应用完整信息。

页面组件负责业务编排，不把请求逻辑塞入展示组件。管理页优先复用现有 Ant Design Vue 交互范式，不额外抽象通用表格框架。

## 核心业务流程

### 创建与首次生成

1. 用户在主页输入提示词并提交。
2. 未登录时跳转登录页，登录成功后返回原页面；已登录时调用 `POST /app/add`。
3. 获取应用 ID 后跳转 `/app/{id}/chat?autoStart=1`。
4. 对话页加载 `GET /app/get/vo`，把 `initPrompt` 作为首条用户消息。
5. 页面立即移除 `autoStart` 查询参数，再连接 `GET /app/chat/gen/code`，避免刷新页面重复触发生成。
6. SSE `data` 事件中的 JSON `{ "d": "..." }` 追加到当前 AI 消息。
7. 收到 `done` 后关闭连接，把生成状态切换为完成，并加载预览。

### 后续对话

用户提交新消息后，页面添加用户消息和空的 AI 消息，再建立新的 SSE 连接。生成期间禁止重复提交；用户离开页面时主动关闭连接。SSE 失败时保留已收到内容，并提供重试当前消息的操作。

本次后端未提供历史消息接口，因此页面只维护当前会话中的消息；刷新后显示应用初始提示词和可用预览，不伪造历史记录。

### 预览

生成完成后，使用以下地址加载 `iframe`：

`http://localhost:8081/static/{codeGenType}_{appId}/`

实际主机从 `VITE_API_BASE_URL` 获取，默认值沿用 `http://localhost:8081`。每次生成完成时给预览 URL 增加时间戳查询参数，避免浏览器缓存旧页面。预览加载失败时显示可重试空状态，不影响左侧对话。

### 部署

点击部署后调用 `POST /app/deploy`。成功时使用后端返回的 URL，提供“打开网站”和复制地址操作。生成未完成、正在生成或部署请求进行中时禁用部署按钮。

### 列表与管理

- 我的应用调用 `/app/my/list/page/vo`，名称筛选，固定每页不超过 20 条。
- 精选应用调用 `/app/good/list/page/vo`，名称筛选，固定每页不超过 20 条。
- 管理页调用 `/app/admin/list/page/vo`，支持 ID、名称、封面、初始提示词、生成类型、部署标识、优先级和用户 ID 查询。
- 管理页“精选”调用管理员更新接口，将优先级设为 `99`，成功后原位刷新列表。
- 普通用户删除自己的应用使用 `/app/delete`；管理员删除任意应用使用 `/app/admin/delete`。

## 编辑权限

编辑页先读取应用详情。管理员使用 `/app/admin/get/vo`，普通用户使用 `/app/get/vo`。

- 普通用户只显示并提交应用名称。
- 管理员可编辑应用名称、封面 URL 和优先级。
- 普通用户访问他人应用编辑页时，即使公开详情接口返回数据，也必须在前端通过 `userId` 进行拦截；后端更新接口继续作为最终权限边界。
- 保存成功后返回来源页面；没有来源信息时返回主页。

## 状态与错误处理

- 所有列表分别维护加载、空数据、错误和分页状态，互不阻塞。
- 创建、发送、部署、保存和删除使用独立提交状态，防止重复请求。
- `40100` 跳转登录，`40300` 返回首页并显示无权限提示，其他错误显示后端消息。
- 删除使用确认对话框；精选为可恢复的普通更新操作，不增加二次确认。
- SSE、预览和部署错误在对应操作区域展示，不用全局错误覆盖整个页面。
- 所有外部地址在新标签页打开时使用 `noopener,noreferrer`。

## 测试策略

### 单元测试

- 应用查询参数清理、分页上限和预览 URL 构建。
- SSE 数据解析、`done` 事件、异常关闭和主动关闭。
- 权限判断：管理员、本人、未登录和他人应用。
- 主页创建成功后的路由参数和首次自动生成去重逻辑。

### 组件与页面验证

- 首页列表的加载、空状态、搜索和翻页。
- 对话页流式消息追加、生成完成后的预览刷新、重复发送保护。
- 编辑页普通用户和管理员字段差异。
- 管理页查询、精选和删除后的列表刷新。

### 最终验收

- 运行前端单元测试、类型检查、构建和 lint。
- 启动本地前端与现有 `8081` 后端，使用桌面和移动视口逐页检查。
- 验证无横向溢出、文本遮挡、空白 iframe、重复 SSE 请求和未关闭连接。

## 完成标准

- 四个页面及对应路由均可访问，权限行为正确。
- 创建、SSE 生成、预览与部署形成完整可用链路。
- 我的应用、精选应用和管理员列表均支持规定的查询与分页。
- 普通用户和管理员的编辑、删除权限符合需求。
- 不修改或覆盖用户当前未提交的后端改动。
- 自动测试、类型检查和生产构建通过，关键页面完成桌面与移动端视觉验收。
